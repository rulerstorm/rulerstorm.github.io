Simple GDB

Apr 30, 2017

compile flag

g++ -g -O0 xx.cpp – set optimiaztion level 0

optimization often cause problem for debuggers

list lines

l – list next 10 lines
l n, m – list from line n to m
l foo.cpp:100 — you can specify the file name
l foo.cpp:aFunc — you can specify the function name

execution program

run – run
run arg1 agr2 – specify command line agrugments
s – step into functions
n – step over functions

break points

b $line_number
b $aFunc
tb – temporary breakpoint (disappear after one hit)
info b – list all breakpoints
clear $line_number/$func_name – clear breakpoints
disable $breakpoint_number – note: not the line number!

manipulating data

whatis $var – get the type info of var, even a function
p $var – print it
p some_expression – even a function call
set var $var=$value – modify the value of an variable

monitoring variables

display $expression – print this expression in every breakpoint
info display – display infomations
watch $expression – break when the value of expression is changed, powerful!!
info locals – all local variables

core dump

shell:
ulimit -a – get all of the ulimit info
ulimit -c unlimited – set core dump size
GDB:
bt – back trace, then you can info locals and so on.

other notes

  1. info – show the list of info, including proc, vtbl, threads
  2. - – GUI mode
